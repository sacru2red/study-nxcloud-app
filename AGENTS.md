## 규칙

- 모든 파일의 네이밍은 kebab case로 한다.
- any 타입은 사용하지 않는다. 필요한 경우 작성자에게 허가를 받고 사용한다.

## UI 작업

- UI 관련 작업(컴포넌트, 스타일, 레이아웃, 디자인 토큰 등)을 시작하기 전에 반드시 `DESIGN.md`를 읽고 확인한다.
- 색상, 타이포그래피, 간격, border-radius, 컴포넌트 스펙 등은 `DESIGN.md`에 정의된 토큰과 가이드를 따른다. 임의로 hex/px 값을 새로 정의하지 않는다.
- `DESIGN.md`에 없는 UI 요소를 추가해야 할 때는, 기존 토큰·패턴과 일관되게 확장하고 필요하면 `DESIGN.md` 업데이트를 제안한다.
